    1. Create `packages/bot-harness/training/tui/types.ts` with these interfaces:

       ```typescript
       import type { TrainingLogEntry } from '../training-logger.js';
       import type { SelfPlayConfig } from '../training-config.js';

       export interface TrainingProgressData {
         entry: TrainingLogEntry;
         generation: number;        // batch/generation number (increments per PPO update)
         totalEpisodes: number;
         completedEpisodes: number;
         poolSize: number;
         latestCheckpointEpisode: number | null;
         selfPlayConfig: SelfPlayConfig;
         startTime: number;         // Date.now() at training start
         paused: boolean;
         generationStartTime: number; // Date.now() at start of current generation
         generationEpisodeCount: number; // episodes in current generation/batch
       }

       export type TrainingProgressCallback = (data: TrainingProgressData) => void;

       export interface DashboardState {
         currentData: TrainingProgressData | null;
         rewardHistory: number[];
         policyLossHistory: number[];
         valueLossHistory: number[];
         winRateHistory: number[];
         entropyHistory: number[];
         recentEpisodes: TrainingLogEntry[];
         showHelp: boolean;
         activeView: number;        // index of the current detail view (for Tab cycling)
       }
       ```

    2. Modify `TrainingCoordinator` in `training-coordinator.ts`:

       a. Add import: `import type { TrainingProgressData, TrainingProgressCallback } from './tui/types.js';`
       b. Add public properties to the class:
          ```typescript
          public onProgress: TrainingProgressCallback | null = null;
          private paused: boolean = false;
          private stopRequested: boolean = false;
          private generationCounter: number = 0;
          ```
       c. In the `run()` method, add a pause check at the top of the while loop:
          ```typescript
          // D-12: Pause check
          while (this.paused && !this.stopRequested) {
            await new Promise(resolve => setTimeout(resolve, 100));
          }
          if (this.stopRequested) break;
          ```
       d. After the `await this.logger.logEpisode(entry)` call (inside the `for (const result of episodeResults)` loop), fire the callback:
          ```typescript
          if (this.onProgress) {
            this.onProgress({
              entry,
              generation: this.generationCounter,
              totalEpisodes,
              completedEpisodes: this.episodeCounter,
              poolSize: this.opponentPool!.getPoolSize(),
              latestCheckpointEpisode: this.getLatestCheckpointEpisode(),
              selfPlayConfig: this.config.selfPlay,
              startTime,
              paused: this.paused,
              generationStartTime,
              generationEpisodeCount: batchSize,
            });
          }
          ```
       e. Track `generationStartTime` by adding `const generationStartTime = Date.now();` at the top of the while loop body (before weight extraction), and increment `this.generationCounter++` after the checkpoint block.
       f. Add a `private getLatestCheckpointEpisode(): number | null` helper that parses the latest checkpoint path to extract the episode number, or returns null if no checkpoint exists.
       g. Add public methods:
          ```typescript
          public togglePause(): void {
            this.paused = !this.paused;
          }

          public isPaused(): boolean {
            return this.paused;
          }

          public requestStop(): void {
            this.stopRequested = true;
          }
          ```
       h. In the `cleanup()` method, also set `this.stopRequested = true` to break the loop if running.

    3. Add tests to `training-coordinator.test.ts`:
       - Test that `togglePause()` toggles the `isPaused()` state.
       - Test that `requestStop()` sets the stop flag (verify via a second call to `isPaused()` after stop doesn't change behavior -- this is a basic state test).
       - Test that `onProgress` is null by default.
       - Keep tests minimal since coordinator requires full TF.js init for run(). Focus on the public API surface (togglePause, isPaused, requestStop, onProgress property).
