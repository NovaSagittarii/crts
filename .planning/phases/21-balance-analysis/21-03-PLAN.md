    **clustering.ts** (per D-06):

    Implement `kMeans(data: number[][], k: number, options?: { maxIterations?: number, seed?: number, runs?: number }): ClusterResult`:

    1. **k-means++ initialization**: Select first centroid randomly (seeded). Each subsequent centroid chosen with probability proportional to squared distance from nearest existing centroid. Use seeded LCG PRNG (same as stats.ts) for reproducibility.
    2. **Lloyd's algorithm**: Assign each point to nearest centroid (Euclidean distance). Recompute centroids as cluster means. Repeat until convergence or maxIterations (default 100).
    3. **Multi-run**: If `runs > 1`, run k-means `runs` times with different seed offsets, return the result with lowest WCSS. Default runs=10.
    4. **Edge cases**: k=0 or empty data returns empty result. k > data.length caps k at data.length.
    5. **WCSS computation**: Sum of squared distances from each point to its assigned centroid.

    Helper functions (not exported): `euclideanDistance`, `nearestCentroid`, `kMeansPlusPlusInit`.

    Also export `normalizeFeatures(vectors: StrategyFeatureVector[]): number[][]` — converts StrategyFeatureVector[] to a 2D number array with z-score normalization per feature dimension. This is needed because feature scales differ (buildDensity vs templateEntropy). Returns the normalized matrix. Export separately: `featureVectorToArray(fv: StrategyFeatureVector): number[]` for converting a single vector.

    **sequence-miner.ts** (per D-07):

    Implement `mineSequencePatterns(sequences: string[][], config: { minSupport: number, maxPatternLength: number }): SequencePattern[]`:

    Algorithm: PrefixSpan-style sequential pattern mining.
    1. Find all single-item patterns with support >= minSupport.
    2. For each frequent single-item pattern, recursively extend by one item using projected databases (suffix sequences after each occurrence of the pattern prefix).
    3. Prune when pattern length > maxPatternLength.
    4. Return sorted by support descending, then pattern length descending.

    With only 5 template types, the search space is bounded: at most 5^maxPatternLength candidate patterns. For maxPatternLength=8, that's 390,625 — feasible but PrefixSpan's pruning makes it much faster in practice.

    Also export `extractBuildSequence(match: ParsedMatch, teamId: number): string[]` — extracts the ordered sequence of templateIds from applied build actions for a given team in a match. Skips actions without templateId.

    Use explicit `.js` extensions. Explicit return types on exports. Use seeded PRNG for k-means determinism.
