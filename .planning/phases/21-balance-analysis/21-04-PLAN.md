    **generation-tracker.ts** (per D-13, D-14, D-15):

    `discoverGenerations(checkpointDir: string): Promise<GenerationBoundary[]>`:
    - Read the checkpoints directory with `readdir`.
    - Filter entries matching `checkpoint-*` pattern.
    - Extract episode number by parsing the directory name: `parseInt(name.replace('checkpoint-', ''), 10)`.
    - Sort by episode number ascending.
    - Return array of `{ generation: index+1, episode: number }`.
    - Define `GenerationBoundary` as `{ generation: number, episode: number }` (export it).

    `splitMatchesByGeneration(matches: ParsedMatch[], matchFiles: string[], generations: GenerationBoundary[], checkpointInterval: number): Map<number, ParsedMatch[]>`:
    - Map match index to generation. Match index i belongs to generation g where g is the last generation whose `episode <= matchIndex * checkpointInterval` (or generation 0 for pre-first-checkpoint matches).
    - If no generations provided, all matches are generation 0.
    - Return Map<generationNumber, ParsedMatch[]>.

    `computeGenerationData(generationMatches: ParsedMatch[], generationNumber: number, episode: number, assignments: StrategyAssignment[], config: AnalysisConfig): GenerationData`:
    - Compute template win rates for this generation's matches only.
    - Compute strategy frequency distribution: count of each ruleLabel in the assignments for these matches.
    - Return GenerationData per the type definition.

    **balance-report.ts** (per D-12):

    `assembleBalanceReport(matches: ParsedMatch[], config: AnalysisConfig, options?: { matchDir?: string, checkpointDir?: string }): Promise<BalanceReport>`:
    - This is the main orchestration function that calls all analysis modules:
      1. `classifyAll(matches)` -> StrategyAssignment[]
      2. `normalizeFeatures(assignments.map(a => a.features))` -> number[][]
      3. `kMeans(normalizedData, config.k, { seed: 42 })` -> ClusterResult. Assign clusterId back to assignments.
      4. `computeTemplateWinRates(matches, config)` -> TemplateWinRate[]
      5. `computeStrategyWinRates(matches, assignments, config)` -> StrategyWinRate[]
      6. For each match+team, `extractBuildSequence` -> string[][], then `mineSequencePatterns(sequences, { minSupport: max(2, floor(sequences.length*0.05)), maxPatternLength: config.maxPatternLength })` -> SequencePattern[]
      7. If checkpointDir provided, `discoverGenerations(checkpointDir)` then `splitMatchesByGeneration` then `computeGenerationData` for each generation. Otherwise generations=[].
      8. Assemble into BalanceReport with metadata: { matchDir: options?.matchDir ?? 'unknown', matchCount: matches.length, generatedAt: new Date().toISOString(), confidence: config.confidence }.

    Default AnalysisConfig: `{ confidence: 0.95, minMatches: 10, k: 4, maxPatternLength: 8, firstNBuilds: 3 }`.

    No imports from `training/` modules (only reads files from checkpoint directories). This keeps analysis decoupled from the training runtime (Success Criterion 3, D-15 research pitfall 7).

    ```typescript
    #!/usr/bin/env tsx
    ```

    parseArgs options:
    - `--match-dir <path>` (required) — Directory containing match-*.ndjson files
    - `--run-dir <path>` (optional) — Training run directory for generational tracking. Looks for `<run-dir>/checkpoints/` for generation boundaries.
    - `--output <path>` / `-o` (optional) — Output file path for JSON report (default: stdout for JSON, or `balance-report.json` when `--format all`)
    - `--format <type>` / `-f` (optional) — Output format: `json` | `console` | `markdown` | `all` (default: `console`)
    - `--confidence <float>` (optional, default: 0.95)
    - `--min-matches <int>` (optional, default: 10)
    - `--max-pattern-length <int>` (optional, default: 8)
    - `--k <int>` (optional, default: 4)
    - `--first-n-builds <int>` (optional, default: 3)
    - `--help` / `-h`

    Flow:
    1. Parse args, validate --match-dir exists.
    2. `discoverMatchFiles(matchDir)` -> filePaths.
    3. `readMatchFile` for each file -> ParsedMatch[]. Stream one at a time to avoid memory issues per research anti-pattern.
    4. Build AnalysisConfig from CLI flags.
    5. Determine checkpointDir: if --run-dir provided, use `<run-dir>/checkpoints/`.
    6. `assembleBalanceReport(matches, config, { matchDir, checkpointDir })`.
    7. Output based on --format:
       - `json`: Write JSON.stringify(report, null, 2) to --output or stdout.
       - `console`: Write formatConsoleSummary(report) to stdout.
       - `markdown`: Write formatMarkdownReport(report) to --output or stdout.
       - `all`: Write JSON to `balance-report.json` (or --output), markdown to `balance-report.md`, console summary to stdout.
    8. Print summary line: "Analysis complete: N matches, K templates, M strategies".

    Use `process.exit(0)` on success, `process.exit(1)` on error with stderr message.
