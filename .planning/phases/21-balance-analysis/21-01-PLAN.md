    1. Remove the underscore prefix from `_botActions` and `_teamIds` parameters.

    4. Correlation strategy: For each team, build outcomes arrive in the same order as queued build actions. Group build outcomes by teamId, group build bot actions by teamId, zip them positionally. If counts mismatch (some builds were not queued in this tick), populate what we can and leave the rest optional.

    The fix is backward-compatible: existing fields remain, new fields are populated when bot action data is available.

    NOTE: The TickActionRecord interface already has optional templateId/x/y/transform fields — no type changes needed.
