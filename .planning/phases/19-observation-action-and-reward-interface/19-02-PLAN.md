    **Create `packages/bot-harness/action-decoder.ts`:**

    Export `ActionSpaceInfo` interface:
    ```typescript
    export interface ActionSpaceInfo {
      type: 'Discrete';
      n: number;                    // total action space size (templates * positions + 1)
      numTemplates: number;         // buildable templates (excluding __core__)
      numPositions: number;         // territory position count (upper bound at reset, actual varies)
      templateIds: readonly string[]; // ordered template IDs for index mapping
    }
    ```

    Export `ActionDecoder` class with:
    - Constructor: `(width: number, height: number)` -- grid dimensions for position enumeration bounds
    - Method: `getBuildableTemplates(room: RtsRoom): StructureTemplateSummary[]` -- returns `room.state.templates.filter(t => t.id !== '__core__').map(t => t.toSummary()).sort((a, b) => a.id.localeCompare(b.id))`. Sort alphabetically by id for determinism. This gives 5 templates: block, eater-1, generator, glider, gosper.
    - Method: `enumerateTerritoryPositions(room: RtsRoom, teamId: number): Vector2[]` -- enumerate all grid cells within the build zone. Implementation:
      1. Get team payload via `room.createStatePayload().teams.find(t => t.id === teamId)`
      2. Collect build zone contributors from all own structures with `buildRadius > 0` using `collectBuildZoneContributors()` with `BuildZoneContributorProjectionInput` mapped from `StructurePayload.{x, y, width, height, buildRadius}`
      3. Iterate ALL grid cells (x=0..width-1, y=0..height-1) -- use full grid as upper bound per Pitfall 1 (territory grows as structures are built)
      4. For each cell, check `isBuildZoneCoveredByContributor()` against any contributor
      5. Collect matching cells into array, already in row-major order (y outer, x inner loop)
      6. Return the sorted Vector2[] array

    **IMPORTANT (Pitfall 1 -- Action Space Sizing):** The action space size must be FIXED for the duration of an episode. At reset time, use the FULL GRID (width * height) as `numPositions` upper bound. This means `n = numTemplates * (width * height) + 1`. The mask handles which positions are actually valid each tick. This prevents the issue where territory expansion creates positions with no action index.

    - Method: `computeActionMask(room: RtsRoom, playerId: string, teamId: number): Uint8Array` -- compute per-action validity mask.
      1. Get buildable templates (sorted, excluding __core__)
      2. `maskSize = templates.length * (this.width * this.height) + 1`
      3. Allocate `Uint8Array(maskSize)` initialized to 0
      4. `mask[0] = 1` (no-op always valid)
      5. Get territory positions via `enumerateTerritoryPositions(room, teamId)` -- only iterate these for efficiency
      6. For each territory position, for each template:
         - `actionIdx = templateIdx * (this.width * this.height) + posIdx_in_full_grid + 1`
         - where `posIdx_in_full_grid = pos.y * this.width + pos.x` (row-major index into full grid)
         - Call `room.previewBuildPlacement(playerId, { templateId: template.id, x: pos.x, y: pos.y })`
         - Set `mask[actionIdx] = preview.accepted ? 1 : 0`
      7. Return mask

    **CRITICAL (Pitfall 3):** `previewBuildPlacement` takes `playerId` (string like `'rl-agent'`), NOT `teamId` (number). The room resolves team internally from player.

    - Method: `decode(actionIndex: number): BuildQueuePayload | null` -- decode action integer back to build payload.
      1. If `actionIndex === 0`, return `null` (no-op per D-07)
      2. `adjusted = actionIndex - 1`
      3. `templateIdx = Math.floor(adjusted / (this.width * this.height))`
      4. `posIdx = adjusted % (this.width * this.height)`
      5. `x = posIdx % this.width`, `y = Math.floor(posIdx / this.width)`
      6. Get template id from sorted templates list at `templateIdx`
      7. Return `{ templateId, x, y }`

    - Method: `getActionSpaceInfo(room: RtsRoom): ActionSpaceInfo` -- returns static shape descriptor for Phase 20's PPO network builder per D-16.

    **Key constraints:**
    - Sort templates by `id.localeCompare()` for deterministic ordering
    - Full grid (width * height) as position space -- mask narrows valid set
    - Use `previewBuildPlacement(playerId, ...)` not `previewBuildPlacement(teamId, ...)`
    - No TF.js dependency per D-04
    - Use explicit `.js` extensions in relative imports
