<objective>
Create three runtime-agnostic domain modules in packages/bot-harness that the bot CLI (Plan 3) will consume: model loading with auto-detect, tick budget management with fallback strategies, and payload-based observation encoding that works from socket wire data without requiring an RtsRoom instance.

Purpose: These pure domain modules encapsulate the core logic the live bot needs -- loading a trained model, tracking inference timing against a tick budget, and encoding observations from the RoomStatePayload the bot receives over the socket. Keeping them in packages/ ensures they are testable without Socket.IO.

    - When inferenceMs <= budgetMs, metrics.fallbackTriggered is false
    - When inferenceMs > budgetMs with fallback 'noop', shouldAct() returns false
    - When inferenceMs > budgetMs with fallback 'cached', shouldAct() returns false but lastAction is preserved

    Create `packages/bot-harness/model-loader.ts`:
    - `findLatestModelDir(runsDir: string): Promise<string | null>` -- reads `runsDir` with `readdir`, filters for directories starting with `run-`, sorts descending by name, checks each for `final-model/model.json` existence, returns first match or null.
    - `loadBotModel(options: { modelPath?: string; runsDir?: string }): Promise<tf.LayersModel>` -- if `modelPath` provided, call `loadModelFromDir(modelPath)` directly. If not, call `findLatestModelDir(runsDir ?? 'runs')`, throw if null, then `loadModelFromDir(found)`.
    - Import `loadModelFromDir` from `./training/tfjs-file-io.js`. Use `node:fs/promises` for readdir/stat/access.

    Create `packages/bot-harness/tick-budget.ts`:
    - Export `FallbackStrategy = 'noop' | 'cached' | 'deadline'` type alias.
    - Export `interface TickBudgetConfig { budgetMs: number; fallback: FallbackStrategy; }`.
    - Export `interface TickMetrics { inferenceMs: number; budgetUtilization: number; fallbackTriggered: boolean; fallbackReason?: string; }`.
    - Export `interface TickBudgetStats { totalTicks: number; fallbackCount: number; avgInferenceMs: number; maxInferenceMs: number; }`.
    - Export `class TickBudgetTracker`:
      - Constructor takes `config: TickBudgetConfig` (default `{ budgetMs: 80, fallback: 'noop' }`).
      - `startTick(): void` -- records `performance.now()`.
      - `endTick(): TickMetrics` -- computes elapsed, budgetUtilization = elapsed / budgetMs, fallbackTriggered = elapsed > budgetMs. Updates internal cumulative stats.
      - `shouldAct(metrics: TickMetrics): boolean` -- returns `!metrics.fallbackTriggered`. (For 'cached' fallback, caller handles using lastAction.)
      - `getStats(): TickBudgetStats` -- returns cumulative stats.
      - `formatMetricsLog(metrics: TickMetrics, tick: number): string` -- returns a single-line log string for debugging.

    Tests use filesystem mocking (create temp dirs with `mkdtemp`) for model-loader. Tests use manual timing simulation for tick-budget. Per D-07, verify metrics logging format.

    - Channel 3 (own territory mask) encoded from ownTeam.structures with buildRadius > 0 using collectBuildZoneContributors
    - Channel 4 (own core position) encoded from ownTeam.structures where isCore=true

    - shape.channels=5, shape.scalarCount=7

    Create `packages/bot-harness/payload-observation-encoder.ts`:
    - Import `Grid` from `#conway-core`, `collectBuildZoneContributors`, `isBuildZoneCoveredByContributor`, `DEFAULT_TEAM_TERRITORY_RADIUS` from `#rts-engine`.
    - Import `ObservationResult` from `./observation-encoder.js` (reuse the same interface).
    - `class PayloadObservationEncoder` with constructor `(width: number, height: number)`.
    - `encode(payload: RoomStatePayload, teamId: number, maxTicks: number): ObservationResult`:
      - Find own team and enemy team from `payload.teams`.
      - Channel 0: Reconstruct grid from `payload.grid` using `Grid.fromPacked(new Uint8Array(payload.grid), width, height)`, iterate cells.
      - Channels 1-4: Same logic as `ObservationEncoder` but operating on `TeamPayload` data from `payload.teams`.
      - Scalars 0-5: Same normalization as `ObservationEncoder` but reading from `TeamPayload` fields directly.
      - Scalar 6 (territoryRadius): Compute as `DEFAULT_TEAM_TERRITORY_RADIUS + sum(s.buildRadius for s in ownTeam.structures where !s.isCore and s.buildRadius > 0)`, then normalize by `/100`.
    - The encoder does NOT depend on `RtsRoom` at all -- only on `RoomStatePayload` types and `Grid.fromPacked()`.
