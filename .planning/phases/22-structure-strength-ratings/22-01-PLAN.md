    **Step 2: Create glicko2-engine.ts** implementing Glickman's 8-step algorithm as a pure function.

    Export `GLICKO2_DEFAULTS` object with: `{ initialRating: 1500, initialRd: 350, initialVolatility: 0.06, tau: 0.5, convergenceTol: 1e-6, scaleFactor: 173.7178 }`.

    Export `updateRating(player: Glicko2Rating, matches: MatchResult[], tau?: number): Glicko2Rating`.

    Internal helpers (not exported): `g(phi)`, `E(mu, muJ, phiJ)`, `computeVariance(mu, opponents)`.

    Key implementation details:
    - Scale factor: `173.7178` (which is `400 / ln(10)`)
    - Step 1: Convert to Glicko-2 scale: `mu = (rating - 1500) / 173.7178`, `phi = rd / 173.7178`
    - Step 2: Compute g(phi) = 1/sqrt(1 + 3*phi^2/pi^2), E(mu, muJ, phiJ) = 1/(1 + exp(-g(phiJ)*(mu - muJ)))
    - Step 3: Compute variance v = [SUM g^2 * E * (1-E)]^(-1)
    - Step 4: Compute delta = v * SUM g * (s - E)
    - Step 5: New volatility via Illinois algorithm. Set A = ln(sigma^2), B based on delta^2 vs phi^2+v. Iterate f(x) function until |B-A| < 1e-6. Cap at 100 iterations.
    - Step 6: phi* = sqrt(phi^2 + sigma'^2)
    - Step 7: phi' = 1/sqrt(1/phi*^2 + 1/v), mu' = mu + phi'^2 * SUM g*(s-E)
    - Step 8: Convert back to display scale

    If matches array is empty: only increase RD (Step 6 with no volatility change), return same rating and volatility.

    Use explicit `.js` extension on all relative imports. Use strict TypeScript (no `any`). Add explicit return types on all exported functions.
