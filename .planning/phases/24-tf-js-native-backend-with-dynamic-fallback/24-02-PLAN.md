<objective>
Migrate all TF.js consumer files from direct `import * as tf from '@tensorflow/tfjs'` to use the centralized `getTf()` backend loader.

Purpose: Eliminate all hardcoded TF.js imports so every file gets native acceleration when available, with automatic pure JS fallback. This is the bulk migration that touches 7 production files and 4 test files.

Output: All bot-harness files using getTf(), zero direct value imports of @tensorflow/tfjs remaining.
</objective>

Consumer migration pattern (from RESEARCH.md Pattern 2, Sub-pattern A):
```typescript
// Module-level init + lazy reference approach (RECOMMENDED)
import { getTf } from './tf-backend.js';     // or '../tf-backend.js' for training/ files
import type { TfModule } from './tf-backend.js';
import type * as tf from '@tensorflow/tfjs';  // keep for TYPE annotations only

let _tf: TfModule;

/** Must be called before any other function in this module. */
export async function initTfBackend(): Promise<void> {
  _tf = await getTf();
}

// Rest of file uses `_tf` instead of `tf` for runtime calls
// Type annotations still use `tf.LayersModel`, `tf.Tensor`, etc. from type-only import
```

For test files:
```typescript
import { getTf } from './tf-backend.js';
import type { TfModule } from './tf-backend.js';

let tf: TfModule;

beforeAll(async () => {
  tf = await getTf();
});
```
