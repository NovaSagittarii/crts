<objective>
Create the centralized TF.js backend loader module with dynamic import fallback, unit tests, and package.json configuration.

Purpose: Establish the single entry point (`getTf()`) that all TF.js consumers will use. This replaces direct `import * as tf from '@tensorflow/tfjs'` with a loader that prefers the native C++ backend for 10-30x faster training/inference, falling back transparently to pure JS on Alpine/musl or when the native addon fails.

Output: `tf-backend.ts` module, unit tests, package.json with optionalDependency, barrel export.
</objective>

  <action>
    Create `packages/bot-harness/tf-backend.ts` implementing the centralized backend loader:

    1. Import `type * as tfTypes from '@tensorflow/tfjs'` for the type alias only.
    2. Export `type TfModule = typeof tfTypes` as the canonical type for the TF.js module.
    3. Module-level state: `let _promise: Promise<TfModule> | null = null` and `let _backendName: 'native' | 'cpu' = 'cpu'`.
    4. Private `async function loadBackend(): Promise<TfModule>` that:
       - Tries `const mod = await import('@tensorflow/tfjs-node')` in a try block
       - Sets `_backendName = 'native'` on success, returns `mod as unknown as TfModule`
       - In catch block: `const mod = await import('@tensorflow/tfjs')`, sets `_backendName = 'cpu'`, returns `mod`
    5. Export `function getTf(): Promise<TfModule>` that uses promise-based caching (Pitfall 4 fix):
       - If `_promise === null`, set `_promise = loadBackend()`
       - Return `_promise`
    6. Export `function getBackendName(): 'native' | 'cpu'` returning `_backendName`.

    Use explicit `.js` extension in NO relative imports (this file only imports from npm packages).
    Use explicit return types on all exported functions.
    Do NOT add any console.log/console.error -- the caller can log `getBackendName()` if needed.

    Create `packages/bot-harness/tf-backend.test.ts`:
    1. Import `getTf`, `getBackendName`, `TfModule` from `./tf-backend.js`.
    2. Test "getTf returns a module with tensor, layers, model, train, tidy":
       - `const tf = await getTf()`
       - Assert `typeof tf.tensor === 'function'`
       - Assert `typeof tf.layers === 'object'`
       - Assert `typeof tf.model === 'function'`
       - Assert `typeof tf.train === 'object'`
       - Assert `typeof tf.tidy === 'function'`
    3. Test "getTf returns same instance on repeated calls":
       - `const tf1 = await getTf(); const tf2 = await getTf()`
       - Assert `tf1 === tf2` (referential equality proves caching)
    4. Test "getBackendName returns 'native' or 'cpu' after getTf":
       - `await getTf()`
       - Assert `['native', 'cpu'].includes(getBackendName())`
    5. Test "concurrent getTf calls resolve to same module":
       - `const [a, b, c] = await Promise.all([getTf(), getTf(), getTf()])`
       - Assert `a === b && b === c`

    Use vitest `describe`/`it`/`expect` pattern matching existing test files.
