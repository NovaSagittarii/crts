  <action>
Create `packages/bot-harness/training/ppo-network.ts` implementing per D-01 and D-02:

1. `buildPPOModel(config)` function using TF.js functional API (`tf.model()`):
   - Two inputs: `planeInput` with shape `[height, width, channels]` (channels-last for tf.layers.conv2d) and `scalarInput` with shape `[scalarCount]`. NOTE: ObservationEncoder outputs channels-first `[C, H, W]` but tf.layers.conv2d defaults to channels-last `[H, W, C]`. The network accepts channels-last; callers must transpose. Name inputs `'planes'` and `'scalars'`.
   - CNN trunk: iterate `config.convFilters` (default `[32, 64, 64]`), each layer is `tf.layers.conv2d({ filters, kernelSize: config.convKernelSize, padding: 'same', activation: config.activation, name: 'conv_N' })`.
   - Flatten conv output, concatenate with scalars via `tf.layers.concatenate()`.
   - Shared MLP trunk: iterate `config.mlpUnits` (default `[256, 128]`), each `tf.layers.dense({ units, activation, name: 'trunk_N' })`.
   - Policy head: `tf.layers.dense({ units: actionCount, name: 'policy_logits' })` -- NO activation (raw logits).
   - Value head: `tf.layers.dense({ units: 1, name: 'value' })` -- NO activation (raw scalar).
   - Return `tf.model({ inputs: [planeInput, scalarInput], outputs: [policyLogits, value], name: 'ppo_model' })`.

2. `extractWeights(model)` function: calls `model.getWeights()`, maps each tensor to `{ shape: w.shape, buffer: w.dataSync().buffer.slice(0) }` (cloned ArrayBuffer). Returns `WeightData[]` where `WeightData = { shape: number[]; buffer: ArrayBuffer }`. Disposes intermediate tensors. Per D-17.

3. `applyWeights(model, weightData)` function: creates tensors from WeightData array, calls `model.setWeights(tensors)`, disposes temporary tensors. Per D-17.

4. `PPOModelConfig` interface: `planeShape: [number, number, number]` (channels, height, width -- NOTE this is the observation shape, function transposes internally for network input spec), `scalarCount: number`, `actionCount: number`, `convFilters: number[]`, `convKernelSize: number`, `mlpUnits: number[]`, `activation: string`.

5. Helper `buildModelConfigFromEnv(env: BotEnvironment, networkConfig: NetworkConfig): PPOModelConfig` that reads `env.observationSpace` and `env.actionSpace` to populate the config.

Import TF.js as: `import * as tf from '@tensorflow/tfjs-node'` if tfjs-node was installed successfully, otherwise `import * as tf from '@tensorflow/tfjs'`. The import path should be consistent -- use whichever was verified in Task 1.

Create test file `packages/bot-harness/training/ppo-network.test.ts` with the behaviors listed above. Tests should build a small model (convFilters=[4,8], mlpUnits=[16], small grid like 10x10) to keep tests fast.
