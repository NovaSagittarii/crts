Create `packages/bot-harness/training/convergence.test.ts` -- the key validation gate for Phase 20 success criterion #5.

This test trains a small policy for a short run and asserts it achieves measurably higher win rate against RandomBot than an untrained (random weight) policy.

Implementation:

1. `describe('convergence: trained policy vs untrained baseline')`:

   a. **Test: untrained baseline win rate** -- create a BotEnvironment with small grid (10x10), build a PPO model with tiny config (convFilters=[4,8], mlpUnits=[16]), run 20 evaluation episodes against RandomBot using the untrained model's policy (sample actions from model with random initial weights). Record win count. This establishes the baseline. Expect baseline win rate to be low (roughly random, ~0-30%).

   b. **Test: trained policy exceeds baseline** -- use TrainingCoordinator to train for a short run:
   - Config: totalEpisodes=40, workers=1, gridWidth=10, gridHeight=10, maxTicks=100, convFilters=[4,8], mlpUnits=[16], checkpointInterval=20, batchEpisodes=4, outputDir=tmpDir (use mkdtemp)
   - After training completes, load the final model from `logger.getFinalModelDir()`
   - Run 20 evaluation episodes against RandomBot using the trained model's policy
   - Assert trained win rate > untrained baseline win rate (improvement exists)
   - Assert trained win rate >= 0.55 (55% threshold -- trained policy should beat random play more often than not)
   - If the trained policy doesn't clear 55%, the test fails -- this indicates PPO isn't converging and everything downstream would break

   c. **Cleanup** -- remove temp directory after test

   Set test timeout to 300 seconds (5 minutes). This test is slow due to actual training, so it should be excluded from `npm run test:fast` but included in `npm test`.

   The evaluation function should be a helper `evaluatePolicy(model, envConfig, numEpisodes, opponent)`:
   - For each episode: reset env with opponent, loop stepping with model's sampled actions, record win/loss
   - Return { wins, losses, winRate }

2. Mark this test file with a vitest tag or convention so it can be selectively run:
   ```typescript
   // @vitest-environment node
   // This test performs actual training and takes ~2-5 minutes
   ```

IMPORTANT: Use a small grid and tiny model to keep training fast. The goal is NOT to produce a strong policy -- just to verify the PPO loop converges enough to beat random play with a >55% threshold. If convergence is not observed on 10x10 with 40 episodes, try increasing to 60-80 episodes before declaring failure.
