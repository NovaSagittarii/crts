Create `packages/bot-harness/training/trajectory-buffer.ts`:

1. `TrajectoryStep` interface: `planes: Float32Array` (flat channel-first), `scalars: Float32Array`, `action: number`, `reward: number`, `value: number`, `logProb: number`, `done: boolean`, `actionMask: Uint8Array`. All plain JS types, NO tf.Tensor objects (per research anti-pattern about tensor memory leaks).

2. `computeGAE(rewards, values, dones, lastValue, gamma, lambda)` pure function:
   - `rewards: number[]`, `values: number[]`, `dones: boolean[]`, `lastValue: number`, `gamma: number`, `lambda: number`
   - Returns `{ advantages: Float32Array, returns: Float32Array }`
   - Standard backward GAE pass: for t from T-1 to 0, delta = r[t] + gamma * nextVal * (1 - done[t]) - v[t], gae = delta + gamma * lambda * (1 - done[t]) * prevGae, advantage[t] = gae, return[t] = advantage[t] + v[t].
   - Use `dones[t]` to check if step t is terminal (nextNonTerminal = dones[t] ? 0 : 1).

3. `TrajectoryBuffer` class:
   - `steps: TrajectoryStep[]` array (append via `add(step)`)
   - `advantages: Float32Array | null`, `returns: Float32Array | null` (populated by finalize)
   - `add(step: TrajectoryStep): void` -- push step to array
   - `finalize(lastValue: number, gamma: number, lambda: number): void` -- extract rewards/values/dones arrays from steps, call computeGAE, store results
   - `size(): number` -- returns steps.length
   - `getBatches(batchSize: number): TrajectoryBatch[]` -- shuffles indices, yields mini-batches. Each `TrajectoryBatch` contains typed arrays for planes, scalars, actions, oldLogProbs, advantages, returns, actionMasks for a subset of steps. Advantages are normalized (mean=0, std=1) within the full buffer before batching.
   - `clear(): void` -- resets steps, advantages, returns

4. `TrajectoryBatch` interface: `planes: Float32Array[]`, `scalars: Float32Array[]`, `actions: Int32Array`, `oldLogProbs: Float32Array`, `advantages: Float32Array`, `returns: Float32Array`, `actionMasks: Uint8Array[]`, `size: number`.

Create `packages/bot-harness/training/ppo-trainer.ts`:

1. `TrainStepResult` interface: `policyLoss: number`, `valueLoss: number`, `entropy: number`, `totalLoss: number`, `approxKl: number`.

2. `PPOUpdateResult` interface: `epochsRun: number`, `meanPolicyLoss: number`, `meanValueLoss: number`, `meanEntropy: number`, `meanKl: number`, `earlyStopped: boolean`.

3. `PPOTrainer` class:
   - Constructor takes `model: tf.LayersModel`, `config: TrainingConfig` (uses clipEpsilon, entropyCoeff, valueLossCoeff, learningRate, maxGradNorm, targetKl, ppoEpochs, miniBatchSize).
   - Creates `tf.train.adam(config.learningRate)` optimizer internally.
   - `trainOnBatch(batch: TrajectoryBatch): TrainStepResult` -- runs a single gradient update:
     a. Convert batch arrays to tensors inside `tf.tidy()`.
     b. For planes: each step's flat Float32Array `[C*H*W]` needs to be reshaped to `[H, W, C]` (channels-last). Stack all steps into `[batchSize, H, W, C]` tensor. Get H, W, C from the model's input shape.
     c. Stack scalars into `[batchSize, scalarCount]`.
     d. `optimizer.minimize(() => { ... })` with clipped surrogate loss:
        - Forward pass: `[logits, values] = model.predict([planesTensor, scalarsTensor])`
        - Squeeze values from `[batch, 1]` to `[batch]`
        - Compute `logSoftmax(logits)`, gather new log probs for taken actions using one-hot multiplication + sum
        - Apply action mask: add `(1 - mask) * -1e9` to logits before logSoftmax (per research Pitfall 5)
        - Compute ratio = exp(newLogProb - oldLogProb)
        - Clipped ratio = clip(ratio, 1-eps, 1+eps)
        - Policy loss = -mean(min(ratio * advantage, clippedRatio * advantage))
        - Value loss = mean((predictedValue - returns)^2)
        - Entropy = -mean(sum(probs * logProbs, axis=-1))
        - Total loss = policyLoss + valueLossCoeff * valueLoss - entropyCoeff * entropy
        - Compute approxKl = mean((ratio - 1) - log(ratio)) for early stopping
     e. Return TrainStepResult with all loss components.

   - `update(buffer: TrajectoryBuffer): PPOUpdateResult` -- runs `ppoEpochs` epochs:
     a. For each epoch, get shuffled mini-batches from buffer via `buffer.getBatches(miniBatchSize)`
     b. Run `trainOnBatch` on each mini-batch, accumulate results
     c. If mean approxKl for the epoch exceeds `targetKl`, stop early (per research Pitfall 6)
     d. Return PPOUpdateResult with aggregated metrics

   - `sampleAction(logits: tf.Tensor1D, actionMask: Uint8Array): { action: number, logProb: number }`:
     a. Inside `tf.tidy()`: apply mask by adding `(1 - mask) * -1e9` to logits
     b. Sample using `tf.multinomial(maskedLogits.expandDims(0), 1)`
     c. Compute logProb via `tf.logSoftmax(maskedLogits)` gathered at sampled action index
     d. Return action index and log prob as plain numbers

   - `computeValue(planes: Float32Array, scalars: Float32Array): number`:
     a. Inside `tf.tidy()`: reshape planes to [1, H, W, C], stack scalars to [1, scalarCount]
     b. Forward pass, return value head output as plain number

   - `getOptimizerWeights(): Promise<tf.NamedTensor[]>` -- for resume support per D-11
   - `setOptimizerWeights(weights: tf.NamedTensor[]): Promise<void>` -- for resume support per D-11

CRITICAL: Wrap ALL tensor operations in `tf.tidy()` to prevent memory leaks. Tensors returned from tidy must be the return value. Use `tf.keep()` only when intentionally persisting a tensor.

Create test file using a small model (4x4 grid, 2 conv filters, 8 MLP units) and synthetic trajectory data. Tests verify loss decreases, KL early stopping works, no tensor leaks.
