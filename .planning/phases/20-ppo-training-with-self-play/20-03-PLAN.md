  <action>
Create `packages/bot-harness/training/opponent-pool.ts`:

1. `OpponentType` union type: `'latest' | 'historical' | 'random'`.

2. `OpponentEntry` interface: `type: OpponentType`, `name: string`, `path: string | null` (null for built-in bots), `episode: number` (episode when checkpoint was created, -1 for built-in), `strategy: BotStrategy | null` (cached for built-in bots).

3. `OpponentPool` class:
   - Constructor takes `config: SelfPlayConfig`, `checkpointBaseDir: string` (e.g., `runs/run-id/checkpoints`).
   - Initializes `builtInBots: OpponentEntry[]` with RandomBot and NoOpBot entries (per D-08). These have `type: 'random'`, `strategy` set to instances, `path: null`.
   - `checkpoints: OpponentEntry[]` array for historical model checkpoints.
   - `latestCheckpointPath: string | null` -- tracks the most recent checkpoint.

   - `sampleOpponent(): OpponentEntry` -- draws from the three-way ratio (per D-05):
     a. Generate random number [0, 1)
     b. If < latestRatio and latestCheckpointPath exists: return entry for latest checkpoint
     c. Else if < latestRatio + historicalRatio and checkpoints.length > 0: return random historical checkpoint
     d. Else: return random built-in bot (RandomBot or NoOpBot)
     e. If the selected category is empty, fall back to the next available category

   - `shouldCheckpoint(episode: number): boolean` -- returns `episode > 0 && episode % config.checkpointInterval === 0` (per D-06)

   - `async addCheckpoint(path: string, episode: number): Promise<void>`:
     a. Create entry with `type: 'historical'`, `path`, `episode`, `strategy: null`
     b. Push to checkpoints array
     c. Update `latestCheckpointPath = path`
     d. If `checkpoints.length > config.maxPoolSize`, shift (remove oldest) per D-07 FIFO eviction

   - `async saveCheckpoint(model: tf.LayersModel, episode: number): Promise<string>`:
     a. Compute checkpoint directory: `${checkpointBaseDir}/checkpoint-${episode}`
     b. `await mkdir(dir, { recursive: true })`
     c. `await model.save('file://' + dir)` (per D-04, TF.js SavedModel format)
     d. Call `addCheckpoint(dir, episode)`
     e. Return directory path

   - `async loadOpponentWeights(entry: OpponentEntry): Promise<WeightData[] | null>`:
     a. If entry.path is null (built-in bot), return null
     b. Load model via `tf.loadLayersModel('file://' + entry.path + '/model.json')`
     c. Extract weights via `extractWeights(loadedModel)`
     d. Dispose loaded model
     e. Return weights

   - `getLatestCheckpointPath(): string | null`
   - `getPoolSize(): number` -- returns checkpoints.length
   - `getBuiltInBot(entry: OpponentEntry): BotStrategy` -- returns cached strategy or throws

Create test file. For sampling ratio tests, use a seeded Math.random replacement or run 1000 samples and check proportions within 10% tolerance (e.g., 'latest' between 40-60% of 1000). For FIFO eviction, use maxPoolSize=3 and add 5 checkpoints, verify only latest 3 remain. For shouldCheckpoint, test with interval=50 at episodes 0, 49, 50, 100.

NOTE: Tests that save/load models to disk should use a temp directory and clean up after. Use `node:fs/promises mkdtemp` for isolation.
