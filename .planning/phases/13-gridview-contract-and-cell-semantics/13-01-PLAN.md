<context>
Phase 13 freezes transformed cell semantics for downstream geometry consumers. Current transform code is function-heavy and does not expose a dedicated `GridView.cells()` contract with `{ x, y, alive }` for every transformed cell. This plan establishes that contract, preserves deterministic ordering, and hard-fails on duplicate coordinates.

Per updated team guidance, if transformed-cell logic needs many tightly related methods, those methods should be co-located in an OOP object/class to improve readability and maintainability.
</context>
