<objective>
Wire broadcast suppression for input-only lockstep mode (XPORT-01), integrate the InputEventLog ring buffer into server room lifecycle (XPORT-02), update the client to stop requesting state:grid during active simulation, and create integration tests proving all three requirements.

Purpose: This is the core transport protocol change -- switching active match traffic from full-state broadcasts to input-only relay with periodic hash checkpoints. After this plan, steady-state lockstep matches send only `build:queued`/`destroy:queued` events and `lockstep:checkpoint` hashes.

Output: Modified server.ts with broadcast gating, modified client.ts checkpoint handler, new integration test file.
