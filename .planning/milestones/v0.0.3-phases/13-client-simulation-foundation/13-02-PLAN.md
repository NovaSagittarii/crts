<objective>
Create the `ClientSimulation` module in `apps/web/src/` that owns the client's local `RtsRoom`, subscribes to server events to replay inputs, advances on server-driven tick signals, and exposes state for rendering. This is the client-side half of the lockstep protocol.

Purpose: The client needs a local simulation runner that stays in sync with the server. Without it, the client depends on full state broadcasts for every tick. `ClientSimulation` is the bridge between socket events and the local sim.

Output: `apps/web/src/client-simulation.ts` module; `tests/web/client-simulation.test.ts` unit tests.
</objective>

    ```typescript
    export type ClientSimulationStatus = 'idle' | 'initialized' | 'running';

    export class ClientSimulation {
      private rtsRoom: RtsRoom | null = null;
      private _currentTick: number = 0;
      private _status: ClientSimulationStatus = 'idle';

      // --- Lifecycle ---
      get status(): ClientSimulationStatus;
      get isActive(): boolean;  // status !== 'idle'
      get currentTick(): number;
      get currentState(): RoomState | null;

      initialize(
        payload: RoomStatePayload,
        templates: StructureTemplate[],
      ): void;

      // --- Tick Advance (server-driven, SIM-02) ---
      advanceToTick(targetTick: number): void;

      // --- Input Replay ---
      applyQueuedBuild(payload: BuildQueuedPayload): void;
      applyQueuedDestroy(payload: DestroyQueuedPayload): void;

      // --- Verification ---
      verifyCheckpoint(serverCheckpoint: RoomDeterminismCheckpoint): boolean;
      createLocalCheckpoint(): RoomDeterminismCheckpoint | null;

      // --- Cleanup ---
      destroy(): void;
    }
    ```

    **Implementation details:**

    `initialize(payload, templates)`:
    - Set `this.rtsRoom = RtsRoom.fromPayload(payload, templates)`
    - Set `this._currentTick = payload.tick`
    - Set `this._status = 'initialized'`

    `advanceToTick(targetTick)`:
    - Guard: if `!this.rtsRoom` or `targetTick <= this._currentTick`, return early
    - Loop: `while (this._currentTick < targetTick) { this.rtsRoom.tick(); this._currentTick = this.rtsRoom.state.tick; }`
    - If `this._status === 'initialized'`, set to `'running'` on first tick advance

    `applyQueuedBuild(payload)`:
    - Guard: if `!this.rtsRoom`, return
    - Get the team: `const team = this.rtsRoom.state.teams.get(payload.teamId)`
    - If no team, log warning and return
    - Look up template: `const template = this.rtsRoom.state.templateMap.get(payload.templateId)`
    - Force-insert the `BuildEvent` directly into `team.pendingBuildEvents` (bypass `queueBuildEvent` validation since server already confirmed):
      ```typescript
      const buildEvent: BuildEvent = {
        id: payload.eventId,
        teamId: payload.teamId,
        playerId: payload.playerId,
        templateId: payload.templateId,
        x: payload.x,
        y: payload.y,
        transform: payload.transform,
        executeTick: payload.executeTick,
        reservedCost: template?.activationCost ?? 0,
      };
      team.pendingBuildEvents.push(buildEvent);
      ```
    - Deduct resources: `team.resources -= buildEvent.reservedCost` (mirrors server queue acceptance behavior)

    `applyQueuedDestroy(payload)`:
    - Guard: if `!this.rtsRoom`, return
    - Get the team: `const team = this.rtsRoom.state.teams.get(payload.teamId)`
    - If no team, log warning and return
    - If `payload.idempotent`, check if the destroy event already exists (same eventId) and skip if so
    - Force-insert the `DestroyEvent` directly into `team.pendingDestroyEvents`:
      ```typescript
      const destroyEvent: DestroyEvent = {
        id: payload.eventId,
        teamId: payload.teamId,
        playerId: payload.playerId,
        structureKey: payload.structureKey,
        executeTick: payload.executeTick,
      };
      team.pendingDestroyEvents.push(destroyEvent);
      ```

    `verifyCheckpoint(serverCheckpoint)`:
    - Guard: if `!this.rtsRoom`, return false
    - `const local = this.rtsRoom.createDeterminismCheckpoint()`
    - Return `local.hashHex === serverCheckpoint.hashHex`

    `createLocalCheckpoint()`:
    - Guard: if `!this.rtsRoom`, return null
    - Return `this.rtsRoom.createDeterminismCheckpoint()`

    `destroy()`:
    - Set `this.rtsRoom = null`
    - Set `this._currentTick = 0`
    - Set `this._status = 'idle'`

    **Use Option A** because it handles both fresh match starts and mid-match joins (reconnects).
